# 期货行业智能资讯监控平台上线与维护说明

## 一、上线方式

第一版采用 GitHub Pages 免费方案：

- 前端部署到 GitHub Pages。
- 网站可使用 GitHub 自动生成的 `github.io` 免费地址。
- 不需要购买服务器。
- 不需要购买域名。
- GitHub Actions 每 30 分钟自动抓取公开来源，并生成 `public/data/live.json`。

## 二、部署前需要用户提供什么

只需要一次性准备：

1. 一个 GitHub 账号。
2. 本机已登录 GitHub CLI。
3. 执行仓库创建、推送，并开启 GitHub Pages。

如果暂时不部署，本项目仍可本地运行和验证。

## 三、日常是否需要维护

正常情况下不需要日常维护。系统会自动完成：

- 自动抓取公开来源；
- 自动识别来源权威性；
- 自动清洗、去重和降权；
- 自动分类和重要性排序；
- 自动生成领导简报和员工资讯；
- 自动显示运行状态；
- 抓取失败时显示系统状态和“暂无确认数据”，不会使用虚构新闻填充页面。

## 四、免费方案边界

- 适合个人主用、部门小范围阅读、领导简报辅助。
- 不适合替代金融终端或实时行情系统。
- 不绕过付费资讯源、账号权限和网站访问限制。
- 免费平台不承诺企业级 SLA。

## 五、后续可选增强

- 增加访问口令或 Cloudflare Zero Trust。
- 增加更多公开 RSS/API 数据源。
- 接入公司内部允许使用的数据源。
- 增加 D1/KV 长期归档。
- 如后续需要无人访问也自动预热，可在 Cloudflare 中增加 Cron Trigger。
- 增加 AI 大模型摘要接口。
- 增加移动端简报推送。
