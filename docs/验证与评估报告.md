# 验证与评估报告

## 一、评估目标

本项目按“免费上线、低维护、主要供个人管理使用、兼顾领导和员工阅读”的目标进行设计和验证。

## 二、已覆盖能力

- 我的资讯工作台；
- 领导简报视图；
- 员工资讯视图；
- 监管与交易所；
- 同行动态；
- 金融要闻；
- 品种异动；
- 简报中心；
- 系统状态；
- 来源权威性评分；
- 内容自动分类；
- 重复资讯合并；
- 重要性排序；
- GitHub Pages 免费部署配置；
- GitHub Actions 定时抓取与静态数据生成。

## 三、数据处理评估

系统采用规则引擎自动处理公开来源资讯：

1. 来源权威性：按监管机构、交易所、协会、政府部门、主流媒体、行业参考、待验证来源分层。
2. 分类：按监管与交易所、同行动态、金融要闻、品种异动、产业资讯分类。
3. 去重：同标题或高度相似标题保留评分更高来源。
4. 重要性：根据来源权威性、业务影响、风险关键词、品种关键词自动排序。
5. 兜底：实时抓取不足时显示“暂无确认数据”，不使用虚构新闻填充页面。

## 四、验收标准

上线前后建议按以下标准评估：

- 页面可正常打开；
- 手机和电脑端主要内容不重叠；
- 今日重点、领导简报、员工资讯均有内容；
- 每条资讯展示来源、时间、重要性、摘要、原因和原文链接；
- 系统状态页显示最近更新和抓取情况；
- 单个来源失败不影响页面展示；
- 未确认真实来源的内容不得进入资讯列表；
- 复制简报功能可用；
- 自动分类和重要性排序通过测试用例。

## 五、当前限制

- 当前未绑定付费资讯源，也不会绕过任何付费或登录限制。
- 免费公开网页结构可能变化，系统已做兜底，但后续可继续增加更稳定的 RSS/API 源。
- 若未登录 Cloudflare，无法由本机直接完成云端部署。
